**1. New environment variables on SylliFunction (under Environment.Variables):**
```yaml

```

**2. New IAM policies on SylliFunction (under Policies):**
```yaml
- S3WritePolicy:
    BucketName: sylli-materials-bucket
- S3ReadPolicy:
    BucketName: sylli-materials-bucket
- DynamoDBWritePolicy:
    TableName: sylli-materials-table
- DynamoDBReadPolicy:
    TableName: sylli-materials-table
- Statement:
    - Effect: Allow
      Action:
        - lambda:InvokeFunction
      Resource: !GetAtt EmbedWorkerFunction.Arn
```

**3. New EmbedWorkerFunction Lambda resource:**
```yaml
EmbedWorkerFunction:
  Type: AWS::Serverless::Function
  Properties:
    CodeUri: backend/
    Handler: workers.embed_worker.lambda_handler
    Runtime: python3.13
    Architectures:
      - arm64
    Timeout: 300
    Environment:
      Variables:
        MATERIALS_BUCKET: sylli-materials-bucket
        MATERIALS_TABLE: sylli-materials-table
        VECTOR_BUCKET_NAME: sylli-vectors
        VECTOR_INDEX_NAME: materials-index
        AWS_REGION_NAME: us-east-1
    Policies:
      - S3ReadPolicy:
          BucketName: sylli-materials-bucket
      - DynamoDBWritePolicy:
          TableName: sylli-materials-table
      - DynamoDBReadPolicy:
          TableName: sylli-materials-table
      - Statement:
          - Effect: Allow
            Action:
              - bedrock:InvokeModel
            Resource: "*"
          - Effect: Allow
            Action:
              - s3vectors:PutVectors
            Resource: "*"
```

Note: s3vectors IAM actions use the prefix `s3vectors:*`. Use `s3vectors:PutVectors` specifically for the embed worker.

**4. New MaterialsBucket S3 resource:**
```yaml
MaterialsBucket:
  Type: AWS::S3::Bucket
  Properties:
    BucketName: sylli-materials-bucket
```

**5. New MaterialsTable DynamoDB resource:**
```yaml
MaterialsTable:
  Type: AWS::DynamoDB::Table
  Properties:
    TableName: sylli-materials-table
    BillingMode: PAY_PER_REQUEST
    AttributeDefinitions:
      - AttributeName: material_id
        AttributeType: S
      - AttributeName: user_id
        AttributeType: S
      - AttributeName: uploaded_at
        AttributeType: S
    KeySchema:
      - AttributeName: material_id
        KeyType: HASH
    GlobalSecondaryIndexes:
      - IndexName: user_id-index
        KeySchema:
          - AttributeName: user_id
            KeyType: HASH
          - AttributeName: uploaded_at
            KeyType: RANGE
        Projection:
          ProjectionType: ALL
```

CRITICAL: The GSI on user_id + uploaded_at MUST be declared here. DynamoDB GSIs cannot be added to an existing table without recreation — if omitted now, the list-materials endpoint will require a full table scan.
