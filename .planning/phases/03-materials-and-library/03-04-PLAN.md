Status polling pattern (from 03-RESEARCH.md):
```typescript
function pollEmbedStatus(materialId: string, token: string, onDone: (status: string) => void) {
  const interval = setInterval(async () => {
    const res = await apiFetch(`/api/v1/materials/${materialId}/status`)
    if (res.ok) {
      const data = await res.json()
      if (data.embed_status === "ready" || data.embed_status === "error") {
        clearInterval(interval)
        onDone(data.embed_status)
      }
    }
  }, 4000)
  return interval  // caller clears on unmount
}
```

State managed by this component:

- `selectedWeek`: the week number to confirm (initialized from pendingMaterial.week_number)
- `showWeekDropdown`: boolean controlling the change-week dropdown

**Upload flow (RAW fetch — NOT apiFetch):**
```typescript
const token = localStorage.getItem("token")
const formData = new FormData()
formData.append("file", file)
const url = syllabusId
  ? `${API_BASE}/api/v1/materials?syllabus_id=${syllabusId}`
  : `${API_BASE}/api/v1/materials`
const res = await fetch(url, {
  method: "POST",
  headers: { Authorization: `Bearer ${token}` },
  body: formData,
  // Do NOT set Content-Type — browser sets multipart boundary automatically
})
```

On upload success: set pendingMaterial from response JSON, set selectedWeek from response.week_number.

**Inline confirmation UI (shown when pendingMaterial is non-null):**
- Show: "'{filename}' → Week {N}: {topic}" with Confirm and Change week buttons
- "Change week" button toggles `showWeekDropdown`
- Dropdown: `<select>` with one `<option>` per week from weekMap.weeks, showing "Week {N}: {topic}"
  - Use input accept attribute: `.pdf,.pptx,application/pdf,application/vnd.openxmlformats-officedocument.presentationml.presentation`
- Changing the dropdown updates selectedWeek immediately (does NOT confirm yet)

**Confirm flow:**
```typescript
const res = await apiFetch(`/api/v1/materials/${pendingMaterial.material_id}/confirm`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ week_number: selectedWeek }),
})
```
On success: start polling, set embedStatus to 'processing'.

**Polling:**
- setInterval every 4000ms calling GET /api/v1/materials/{id}/status
- clearInterval when embed_status === 'ready' OR 'error'
- On 'ready': set embedStatus to 'ready', show checkmark, call onMaterialUploaded(), clear pendingMaterial after 2 seconds
- On 'error': set embedStatus to 'error', show error badge
- Clear interval in useEffect cleanup on unmount

**Status badges:**
- embed_status 'processing': small "Processing..." text in gray
- embed_status 'ready': green checkmark text "Embedded"
- embed_status 'error': red "Embedding failed" text

**File input:** Reset `<input type="file">` after successful upload so user can upload again.

API_BASE should be read from: `const API_BASE = process.env.NEXT_PUBLIC_API_URL ?? "http://localhost:3001"`
