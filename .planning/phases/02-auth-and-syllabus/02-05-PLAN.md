  <what-built>
    Complete Phase 2 auth and syllabus system:
    - Backend: POST /auth/register, POST /auth/login, JWT middleware, user DynamoDB table
    - Backend: Syllabus upload and retrieval gated by JWT, user_id scoped
    - Frontend: Register/login form, AuthContext, dashboard with syllabus upload and week timeline

  <how-to-verify>
    Perform these checks in order. Report which steps pass or fail.

    **1. Registration**
    - Open http://localhost:3000 in the browser
    - You should be redirected to /login
    - Click "Register" to switch to registration mode
    - Enter a username (min 3 chars) and a 4-8 digit PIN
    - Click "Create account"
    - Expected: redirected to /dashboard, your username appears in the header

    **2. Syllabus upload**
    - On the dashboard, click the upload area and select a PDF syllabus
    - Expected: upload completes and the week/unit timeline appears below the upload area
    - Confirm the course name and at least one week entry are visible

    **3. Logout and re-login**
    - Click "Sign out"
    - Expected: redirected to /login
    - Sign in with the same username and PIN
    - Expected: redirected to /dashboard (token stored in localStorage and auth works)

    **4. Cross-user isolation (AUTH-02)**
    - In a new incognito window, go to http://localhost:3000 and register a DIFFERENT username
    - Then try this curl (replace SYLLABUS_ID with the first user's syllabus_id from the Network tab):
      ```
      curl -H "Authorization: Bearer TOKEN_FROM_USER_2" \
           http://127.0.0.1:3001/Prod/api/v1/syllabus/SYLLABUS_ID_FROM_USER_1
      ```
    - Expected: 404 {"detail": "Syllabus not found."} — user 2 cannot access user 1's data

    **5. Unauthenticated access blocked**
    - curl http://127.0.0.1:3001/Prod/api/v1/syllabus/anything-here (no Authorization header)
    - Expected: 403 {"detail": "Not authenticated"} (FastAPI HTTPBearer returns 403 for missing header)
  </how-to-verify>
  <resume-signal>Type "approved" if all 5 checks pass. Or describe which checks failed and what you observed.</resume-signal>
